## 1. 背景与问题

### 1.1 现状

"目标地图"是从商用目标地图 Excel 的"全量进度管理"页迁移而来的在线视图，仅登录用户可见，用于对内/对上展示产品能力全貌与未来规划。当前实现（`实现方式/index.html` 的 `tmCardHtml`）为：**按场景分组，把每一条需求渲染成一张完全展开的卡片**——标题 + 5 个并排文字标签（环节/等级/满足度/状态/业务线）+ 现状全文 + 后续计划全文。

### 1.2 核心问题

页面信息密度过高，阅读体验反而不如原始 Excel。根因有三：

1. **丢失层级**：源数据是 `场景 → 环节 → 功能 → 需求` 四级树，Excel 用合并单元格保留了层级。现页面只按 `场景` 分组，`环节` 被降级为标签、`功能` 未展示，同一场景下几十条需求平铺，读者失去"身处树中何处"的定位。
2. **丢失对齐**：卡片内标签流式排列、长度不一，现状/后续计划为整段文字，相邻条目无法在同一列上对比"满足度""状态"。Excel 一列扫读 3 秒可完成的对比，卡片需逐张精读。
3. **没有主次**：每条都把"现状/后续计划"整段铺在总览位置，把"逐条精读"的信息放到了"快速总览"的场景，缺少可扫读的全局视图。



### 1.3 目标

- **一眼看出成熟度**：进入页面即可通过颜色分布感知各场景/环节的能力强弱与空白区。
- **保留精准查阅**：需要逐条核对满足度与排期时，能像 Excel 一样对齐、可筛选、可展开明细。
- **降低视觉密度、保持信息密度**：默认视图用颜色/图标编码替代文字堆叠，长文本按需展开。
- **不改后端**：`/api/public/target-map` 现有返回字段已足够，改动集中在前端渲染与样式。

## 2. 数据模型

数据由 `/api/public/target-map` 返回，单条字段如下（对应 `实现方式/backend/data/target_map_seed.json`）：


| 字段             | 含义   | 取值                                                 | 层级/用途        |
| -------------- | ---- | -------------------------------------------------- | ------------ |
| `scene`        | 场景   | 校本资源建设与管理 / 日常作业与练习 / 教辅作业数据采集 / 多学科统考 / 校级教学管理与洞察 | 一级分组         |
| `link`         | 环节   | 组卷 / 扫描 / 识别与批改 / 数据查看与应用 / 错题的管理与运用 / 异常处理 等      | 二级分组         |
| `func`         | 功能   | 文本（如"多来源兼容""AI自动批阅"）                               | 三级分组 / 明细列   |
| `demand`       | 需求   | 文本                                                 | 叶子节点（最小单元）   |
| `level`        | 等级   | 高 / 中 / 低 / 非必须                                    | 视觉编码（优先级）    |
| `meetStatus`   | 满足度  | 完全满足 / 基本满足 / 完全不满足                                | **核心颜色编码维度** |
| `status`       | 状态   | 开发中 / 已排期 / 待排期 / 暂不响应 / 空                         | 状态点编码        |
| `biz`          | 业务线  | 直播业务 / 录播业务 / 独立项目（可多值，逗号分隔）                       | 筛选维度 / 明细    |
| `currentState` | 现状   | 文本                                                 | 明细（默认收起）     |
| `followUp`     | 后续计划 | 文本                                                 | 明细（默认收起）     |
| `plannedAt`    | 计划时间 | 日期或空                                               | 明细           |


**规模**：共 69 条，分布于 5 个场景（校本资源建设与管理 4 · 日常作业与练习 34 · 教辅作业数据采集 14 · 多学科统考 15 · 校级教学管理与洞察 2）。

## 3. 整体结构

页面自上而下分为四个区域，其中"总览仪表盘""筛选栏""场景目录"为地图/表格两视图共享：

```
┌────────────────────────────────────────────┐
│ 总览仪表盘（KPI + 满足度分布条）  ← 两视图共享  │
├────────────────────────────────────────────┤
│ 视图切换： [ 地图 | 表格 ]                      │
├────────────────────────────────────────────┤
│ 筛选栏（业务线 / 等级 / 满足度 / 状态 / 关键字） │
├───────────────────────────────┬────────────┤
│ 视图主体（地图色块墙 或 层级表格） │ 场景目录（TOC）│
└───────────────────────────────┴────────────┘
```

切换视图时，筛选条件与场景目录保持不变（联动共享一份筛选结果 `matchMap`）。

## 4. 功能需求



### 4.1 总览仪表盘（新增，两视图共享）

**目的**：进入页面 5 秒内抓住整体成熟度。

- **关键指标**（基于当前筛选结果实时计算）：
  - 总需求数
  - 满足度分布：完全满足 N（占比%）· 基本满足 N · 完全不满足 N
  - 状态分布：开发中 N · 已排期 N · 待排期 N · 暂不响应 N
- **满足度分布条**：一条横向堆叠条，绿（完全满足）/ 蓝（基本满足）/ 红（完全不满足），直观显示占比。
- **（可选）按业务线覆盖率**：直播 / 录播 / 独立项目 各自的满足度小条。
- 数据全部由前端对 `targetMapData` 聚合得出，随筛选变化实时刷新。



### 4.2 地图视图（默认视图）

**目的**：用颜色分布呈现能力"地图"，降低视觉密度、突出空白区。

- **分组**：`场景 → 环节` 两级分组。场景为 section，环节为 section 内的小行；`功能`(func) 可作为环节内的次级分组或仅在明细中体现。
- **需求 = 色块**：每条需求渲染为一个小色块，而非展开卡片：
  - **底色 = 满足度**：绿=完全满足、蓝=基本满足、红=完全不满足（复用现有 `--meet-`* 配色）。
  - **状态点**：色块右上角一个小圆点表示状态（开发中/已排期/待排期/暂不响应用不同色；空状态不显示）。
  - **等级**：用角标、边框粗细或不透明度区分 高/中/低/非必须（弱编码，不喧宾夺主）。
  - 色块正文仅显示需求名（1–2 行截断），**不显示** 现状/后续计划/业务线/计划时间。
- **明细下钻**：hover 显示气泡摘要，或点击色块打开右侧抽屉/弹层，展示：现状、后续计划、计划时间、业务线、环节、功能等完整信息。
- **布局**：色块在每个环节行内以 `flex-wrap` 或 `grid` 平铺，保证一屏可见大量色块。
- **空场景**：筛选后无匹配项的场景，标题仍常驻显示（保持地图完整轮廓），下方标注"0 项"。



### 4.3 表格视图

**目的**：忠实还原 Excel 的对齐与扫读优势，供逐条精准核对。

- **列**：`环节 | 功能 | 需求 | 等级 | 满足度 | 状态 | 计划时间 | 业务线`。
- **色点编码**：等级/满足度/状态列用色点或窄色签而非并排文字标签，保证同列上下可瞬间对比。
- **分组行（模拟合并单元格）**：`场景`/`环节` 与上一行相同时不重复显示（留白），保留层级感。
- **明细默认折叠**：表格行默认只显示需求一行；点击行展开一条子行，显示 `现状` + `后续计划` 全文。避免长文本撑高每一行。
- **表头 sticky**：长表滚动时列名常驻。



### 4.4 筛选与场景目录（复用现有）

- 复用现有筛选栏（`filter-targetmap`）：业务线、等级、满足度、状态多选 + 关键字，点击"确定"生效、"清空筛选"重置。
- 关键字检索范围保持：需求 / 现状 / 后续计划 / 业务线。
- 场景目录（TOC）常驻右侧，点击滚动定位到对应场景，需扣减 sticky header 高度。
- 筛选结果同时驱动总览仪表盘、地图视图、表格视图三者。



### 4.5 视图切换

- 顶部提供"地图 / 表格"两个视图切换（复用现有 tab 组件视觉风格）。
- 切换视图时保留当前筛选条件与滚动上下文，不重新拉取数据。
- 默认进入"地图"视图。

## 5. 交互与视觉规范

1. **等级、满足度、状态一律用颜色/图标编码**，不再使用三个并排文字标签（现有密度过高的最大来源）。
2. **现状 / 后续计划默认隐藏**，仅在地图下钻或表格行展开时出现；总览与默认视图不出现长文本。
3. **恢复** `环节` **作为二级分组层级**，并启用当前未使用的 `功能`(func) 字段（表格列 / 地图明细）。
4. 配色沿用现有变量：满足度 `--meet-full/basic/none`，避免引入新色系；状态点色与等级弱编码另行约定但需与满足度色区分。
5. 响应式：窄屏下地图色块自动换行，筛选栏与视图切换保持可用（参考现有 `@media` 断点）。

## 6. 非目标（Out of Scope）

- 不改动后端接口与数据结构（`/api/public/target-map` 返回字段已足够）。
- 不涉及权限模型变更：仍为登录用户可见，不含单校交付明细。
- 不做数据编辑能力（本页为只读展示），编辑仍走后台/Excel 维护流程。
- 不做导出/生图等衍生功能（如有需要另立需求）。

| 阶段  | 内容               | 说明                  |

| P0  | 总览仪表盘            | 改动小、收益最直观，先解决"无全局感" |
